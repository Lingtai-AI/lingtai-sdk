结论

已完成第一刀并通过验证。 五个 LingTai curated MCP（IMAP、Telegram、Feishu、WeChat、WhatsApp）已并入 lingtai kernel wheel，不再依赖独立 addon PyPI wheels。catalog 改以 python -m lingtai.mcp_servers.* 启动；同时保留顶层 lingtai_* 与 lingtai.addons.* 兼容命名空间，避免 TUI importability check 与旧 import 断裂。

改动概要

  • 新增内部实现包：src/lingtai/mcp_servers/{imap,telegram,feishu,wechat,whatsapp}/；保留顶层 src/lingtai_* 兼容 wrappers。
  • 新增兼容 wrappers：src/lingtai/addons/{imap,telegram,feishu,wechat,whatsapp}/，均薄转发到顶层包。
  • pyproject.toml 移除独立 addon wheel 依赖，改为列出第三方 runtime deps：imapclient、lark-oapi、cryptography、qrcode、faster-whisper；并新增六个 console scripts。
  • Feishu 嵌入源码来自已合并事件循环修复后的 lingtai-feishu origin/main，故不再需要等待 lingtai-feishu 0.2.1 PyPI 发布。
  • 更新 MCP manual / curated-addons / troubleshooting 文档，去除“必须先读独立 addon README / 独立 wheel”的主路径表述。
  • 新增回归测试：顶层包 import、lingtai.addons.* 兼容 import、notification header resource、catalog 解压命令均覆盖。

兼容性与风险

  • 兼容：现有 mcp_catalog.json 以 python -m lingtai.mcp_servers.* 启动；旧 lingtai_* 入口仅作兼容。
  • 兼容：lingtai.addons.<name> import 仍可用，服务于旧代码与 TUI addon 检测。
  • 版本：IMAP / Telegram / Feishu / WeChat 的 server version helper 先尝试历史 dist 名，再 fallback 到 lingtai dist 版本，最后 0+local，兼容 editable checkout。
  • 风险：kernel wheel 体积与依赖集合增加，尤其 faster-whisper 依赖较重；这是把 curated MCP 原子化发布的代价。
  • layout fix：Jason requested the embedded servers live as an internal MCP server tree, not as standalone-looking top-level packages. Real implementations now live under lingtai.mcp_servers.*; lingtai_* and lingtai.addons.* are compatibility wrappers only.
  • review fix：DeepSeek review pointed out that WhatsApp's entry point lacked stderr logging. The PR now aligns lingtai_whatsapp.__main__ with the other MCP entry points: logs go to stderr and KeyboardInterrupt is swallowed cleanly. Feishu/Telegram voice dependency error messages now say to reinstall lingtai, not the historical standalone addon wheels.
  • 后续：待该 PR 合并并发布后，再关闭/标记 superseded 旧的 kernel #262 与 TUI #318；不建议在 release/compat 窗口前硬删除历史 PyPI 包。
