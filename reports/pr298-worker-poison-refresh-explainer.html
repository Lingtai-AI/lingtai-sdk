LingTai kernel PR2A implementation note

WorkerStillRunning poison guard + forced refresh recovery

PR2A treats WorkerStillRunningError as a hard process-local poison event: the old in-memory chat interface is no longer safe to mutate, serialize, heal, or reuse. Recovery is durable enough to explain what happened, then moves to a forced refresh/relaunch.

Runtime Semantics

1 Detect The run loop catches WorkerStillRunningError centrally in the AED loop.
2 Poison BaseAgent stores a process-local flag marking the current worker interface unsafe.
3 Persist A bounded redacted JSON artifact is written under history/unfinished_turns/.
4 Recover A high-priority system notification is published and refresh is requested without chat save.

Key Rule

Once poisoned, code must not touch the old ChatInterface. In backend terms, the shared in-memory object may still be owned by another worker thread, so the safe move is to abandon it and rebuild from durable disk state.

no heal no save no wake injection

Poison Guards Added

  • _sync_notifications() skips before notification collection, injection, heal, wake enqueue, and fingerprint commit.
  • _inject_notification_pair() refuses to append synthetic notification pairs when poisoned.
  • _heal_pending_tool_calls(), ASLEEP pre-sleep heal, and legacy tc_inbox drains skip unsafe interface repair.
  • _handle_request() and _handle_tc_wake() return early and request refresh when the current process is poisoned.

Durable Recovery

  • Artifacts use schema version 1 and status open.
  • Request previews are redacted and capped at 500 characters.
  • No raw chat history, tool arguments, or tool results are written.
  • Startup rehydrates the newest open artifact into .notification/system.json if needed.
  • The next safe text request can receive a one-shot kernel recovery notice.

Focused Tests

  • Poisoned _sync_notifications() skips interface mutation and refreshes.
  • Poisoned refresh skips _save_chat_history().
  • _handle_tc_wake() does not heal/save after WorkerStillRunning.
  • Artifact schema and redaction are bounded and do not persist raw unsafe data.
  • Worker-hang system notifications are high priority.

Non-Goals

PR2A does not implement PR2B. Provider cancellation, provider interface isolation, and broader cancellation APIs remain future work. This patch is intentionally smaller: once the current process observes an unsafe worker, it stops touching that interface and asks the existing refresh machinery to rebuild cleanly.

Operational Meaning

"Process-local" means the poison flag lives only in the current Python process. It is not a persisted runtime setting. The persisted part is the unfinished-turn artifact plus the system notification, which explains why the previous turn was abandoned after restart.
