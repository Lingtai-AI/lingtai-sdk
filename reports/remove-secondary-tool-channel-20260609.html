Remove secondary nested tool-call channel

Ready for review Scope: removal only

Summary

This PR removes LingTai's reserved nested secondary tool-call channel. The runtime now keeps tool execution aligned with provider/top-level tool calls: one proposed tool call is executed as one primary tool result, without hidden nested communication dispatch or _secondary.result metadata.

What changed

  • Deleted src/lingtai_kernel/secondary_tools.py.
  • Removed secondary schema injection from base_agent/tools.py; tool schemas still receive reasoning.
  • Simplified ToolExecutor sequential/parallel paths by deleting nested secondary validation/dispatch/result attachment.
  • Added a compatibility guard: stale secondary args from old context/schema are popped before dispatch and logged as deprecated_secondary_ignored; they are never executed.
  • Removed _secondary from spill-manifest hoisting; only current reserved fields such as _duplicate_warning are hoisted.
  • Replaced live prompt/notification guidance with normal top-level read + direct acknowledgement guidance.
  • Updated tests and anatomy docs for the simpler top-level-only model.

Non-goals

  • No tool-call lifecycle ledger in this PR.
  • No approval UI or policy gates.
  • No MCP/Telegram/Email behavior changes beyond removing nested secondary use.
  • No mutation of historical discussion/report files.
