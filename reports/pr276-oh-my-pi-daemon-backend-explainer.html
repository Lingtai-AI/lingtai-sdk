What this adds

A new external-CLI daemon backend for Oh-My-Pi (npm @oh-my-pi/pi-coding-agent, binary omp, repo github.com/can1357/oh-my-pi). Oh-My-Pi's --mode json is a non-interactive JSON event-stream printer that first emits a type:session header (whose id is the resumable session id), then one agent event per JSONL line — structurally an OpenCode-family stream. The implementation therefore reuses the existing OpenCode-family runner and JSON parser rather than duplicating ~190 lines of stream-parsing logic.

How it's wired (minimal, reuse-first)

  • _run_opencode_emanation gained a keyword-only cmd_prefix (default ["run","--format","json"]) alongside the existing executable/backend_name/session_state_key. Oh-My-Pi passes cmd_prefix=["--mode","json","--approval-mode", "yolo"]. OpenCode/MiMo paths are unchanged (default prefix).
  • _run_oh_my_pi_emanation is a thin wrapper over that runner. The OpenCode session-id extractor already recognizes a type:session header with a bare top-level id — no new parser needed.
  • _handle_ask_opencode gained an optional build_resume_cmd(executable, session_id, message) hook. _handle_ask_oh_my_pi supplies _oh_my_pi_resume_cmd for Oh-My-Pi's distinct resume shape; opencode/mimocode keep the default.
  • Dispatch in _handle_emanate_cli and routing in _handle_ask add the oh-my-pi branch; the CLI-backend set and schema enum/description include it.

backend_options reservations

Per-task backend_options stays a passthrough, but Oh-My-Pi's harness-owned flags are rejected pre-flight (whole batch refused with a clear ValueError, no daemon spawned) by _validate_claude_backend_argv:

--mode  --print/-p  --approval-mode yolo  --approval-mode
--session/-r  --resume  --continue/-c  --no-session  --session-dir

These are exactly the flags that drive LingTai's non-interactive JSON harness, session capture, and resume; overriding them would break event parsing or hijack the session. Other scalar/list flags (e.g. --provider, --model) pass through unchanged.

Caveats / risk notes.
  • Built from Oh-My-Pi source research (parseArgs modes + print-mode.ts JSON header shape), not a live omp run; the JSON parser is intentionally permissive and falls back to stdout/stderr if the event vocabulary differs. Smoke-test against an installed omp before relying on ask/resume.
  • --approval-mode yolo is injected for headless operation (approval-mode yolos actions) — appropriate for a disposable daemon worker; flagged here for operator awareness.
  • Resume assumes --session <id> accepts the type:session header id; if a future omp changes that contract, only _oh_my_pi_resume_cmd + the session-id key need updating.
