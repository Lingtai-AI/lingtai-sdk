What the new interface does

from lingtai.core.mcp.licc import push_inbox_event

push_inbox_event(
    "alice",
    "new DM",
    "hey, are you around?",
    metadata={"platform": "telegram"},
)
  • Defaults target from LINGTAI_AGENT_DIR and LINGTAI_MCP_NAME, the env vars the kernel already injects into spawned MCPs.
  • Supports explicit agent_dir / mcp_name for tests and advanced integrations.
  • Writes <agent_dir>/.mcp_inbox/<mcp_name>/<event_id>.json.
  • Uses inbox.validate_event before writing, so the canonical producer only emits receiver-valid LICC v1 events.

Contract alignment

ConcernImplementation
Producer/consumer driftlicc.py re-exports LICC_VERSION, INBOX_DIRNAME, TMP_SUFFIX, and EVENT_SUFFIX from inbox.py.
AtomicityWrites .json.tmp, flushes + fsyncs, then os.replace() to final .json. The poller ignores tmp files.
Listener safetyMissing env, invalid path components, invalid payload fields, filesystem failures, and JSON serialization failures return False; they do not raise into listener callbacks.
Secret hygieneFailure logs do not echo body, subject, or metadata.
ScopeNo addon repo changes, no dependency changes, no standalone package.
